I was commissioned by Dot Web Design to work with Getty Images' API, called Connect. My brief was to design and develop an image search interface for an ecommerce site that sells printed canvas wall panels.

The brief was to make it work like this:

1. The user types in a word or phrase and relevant images appear,

2. The user chooses their favourite image and selects the right size,

3. The user adds the item to the shopping cart and repeats the process until satisfied, then goes to the checkout.

How it was made

I used jQuery for DOM manipulation and to store the input values. The data was then sent using JSON to a pHp script that used the Zend Framework to handle the api call.

Working with the API was challenging because at the time it was quite new and so had a small user community. However Getty does provide quite good documentation and now has a good forum on its api site.

Http Requests with the Zend Framework (v2)

I started by trying to everything in jQuery and JSON, but eventually got stuck when it came to making the http post requests.

In order to make http post requests successfully without falling foul of the same-origin security issues, Getty recommends using the Zend framework. This was my first time using the Zend framework and initially found it quite intimidating due to its overwhelming size. After a few headaches I finally decided that the best approach was to use just the Zend components that I needed.

If anyone is reading this and is having similar problems, please drop me an email and I will send the code I used. Hopefully it will save you a lot of time.
